Winter AI v3.0 -- Polyglot Reasoning Edition -- Created by INEZA Aime Bruno, Rwanda
Backend deployed on Render at https://backend-winter.onrender.com
Frontend deployed on Vercel

Architecture: seven real language layers running as actual subprocess calls or native Python.
Layer 1: Python -- orchestration TF-IDF retrieval language detection
Layer 2: Logic Engine -- pytholog unification PESAD certainty factors acarlson backward chaining
Layer 3: Scheme -- GNU Guile real cond dispatch intent and sentiment classification
Layer 4: Prolog -- SWI-Prolog facts rules backtracking transitive translation
Layer 5: Common Lisp -- SBCL symbolic frame building exact keyword overlap match
Layer 6: OCaml -- compiled native binary algebraic types input validation
Layer 7: C++ -- compiled native binary Levenshtein fuzzy match Unreal Engine payload
Mercury layer: Python re-implementation of Mercury determinism categories

Knowledge folders:
api/info/ -- curated knowledge base files loaded automatically on startup
api/inf/teach/ -- drop your own .txt or .md files here to teach Winter new facts

API endpoints:
POST /api/v1/chats/message -- send prompt get answer plus full reasoning trace
POST /api/v1/reason -- same pipeline trace only for debugging
GET  /api/v1/health -- engine and knowledge status
GET  /api/v1/brain -- read brain.txt
POST /api/v1/brain/update -- overwrite brain.txt
POST /api/v1/teach/upload -- upload a knowledge file into api/inf/teach/
POST /api/v1/teach/reload -- rescan api/info/ and api/inf/teach/
GET  /api/v1/teach/list -- list files loaded from api/inf/teach/
GET  / -- API root info

Teach folder format: plain text or tagged lines like:
EN: your fact in English.
FR: le fait en francais.
RW: ukuri mu Kinyarwanda.
